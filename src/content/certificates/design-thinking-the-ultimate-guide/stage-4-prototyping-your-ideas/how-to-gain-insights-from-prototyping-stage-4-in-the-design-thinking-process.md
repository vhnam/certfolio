title: "How to Gain Insights From Prototyping: Stage 4 in the Design Thinking Process"
---

## Overview

### Role of Prototyping

- Prototyping: A core activity across all stages of design thinking.
- Purpose:
  - Make abstract ideas concrete
  - Enable faster discussion, comparison, and improvement
- Mindset:
  - “Thinking by doing” to unlock deeper insights

## Prototyping for Empathy

### Purpose

- Empathy prototypes: Used in early stages to understand users and problems more deeply.
- Goal: Gain insights into user behaviors, mindsets, and environments.

### Build Prototypes for User Insights

- User-driven prototypes:
  - Ask users to create their own solutions.
  - Focus:
    - Understand user assumptions and desires
- Key idea:
  - Not to use user solutions, but to understand their thinking.

### Build Prototypes for Team Empathy

- Simulation prototypes:
  - Help teams experience user conditions (e.g., poor vision, physical limitations)
- Examples:
  - Vision-impairment simulations
  - “Empathy belly” for pregnancy experience

## Prototyping for Ideation

### Purpose

- Move forward when stuck with many ideas.
- Make ideas tangible to evaluate feasibility and completeness.

### Build Prototypes to Compare Ideas

- Create multiple prototypes:
  - Compare competing solutions
- Benefits:
  - Identify strengths and weaknesses
  - Enable team discussion and idea merging
- Involve:
  - Team members
  - Stakeholders
  - Users

### Tools

- Sketches:
  - Fast and effective for early exploration
  - Help move from abstract → actionable ideas

## Prototyping for Testing

### Purpose

- Validate whether ideas meet user needs.
- Identify major flaws and gaps in understanding.

### Build Prototypes for Stakeholder Feedback

- Test viability:
  - Business feasibility
- Test feasibility:
  - Technical implementation
- Use cases:
  - System diagrams
  - High-fidelity mockups
- Outcome:
  - Gain feedback and stakeholder buy-in

### Build Prototypes for User Testing

- Iterative testing:
  - Build → test → refine repeatedly
- Focus:
  - Answer specific questions with each prototype

### Testing Considerations

- Define questions:
  - Identify what you want to learn
- Context:
  - Test in real or simulated environments
- Fidelity:
  - Match prototype detail to testing goals

## Key Practices

### Guidelines for Effective Prototyping

1. Focus on learning:
   - Each prototype should answer a key question.
2. Stay user-centered:
   - Align with real user needs and behaviors.
3. Iterate continuously:
   - Improve based on feedback.
4. Use appropriate fidelity:
   - Match prototype type to purpose.

## Summary

### Key Takeaways

- Prototypes are versatile tools used across:
  - Empathy
  - Ideation
  - Testing
- Benefits:
  - Gain user insights
  - Compare and refine ideas
  - Validate solutions with users and stakeholders
- Principle:
  - Prototyping is about learning through making.
