## Transition from User Research to Design Models

### Purpose of the Lesson

- Goal: Transform **stories of use** into **structured design models**.
- Objective: Connect **user needs** with **working software solutions**.
- Approach: Use **object-oriented UI design** to align how **users think** with how **developers build systems**.

Key benefit:

- Creates a **clear bridge between research insights and software implementation**.

## Key Concepts Covered

### Three Levels of Detail

- Essential level: Focuses on **core concepts and relationships** in the problem domain.
- Specification level: Adds **attributes, operations, and views** to describe possible solutions.
- Implementation level: Includes **full technical details** needed for development.

Purpose:

- Move gradually from **abstract understanding to concrete solutions**.

### Conceptual Models and UML

- Conceptual models: Represent **key objects and relationships** in a system.
- Unified Modeling Language (UML): A **visual modeling language** used to describe system structure.

Benefits:

- Clarifies how components relate to each other.
- Improves communication between **designers and developers**.

### Stories of Use

- Stories of use: Narratives describing **how users interact with a system to achieve goals**.
- Role in design:
  - Guide conceptual modeling.
  - Ensure the design reflects **real user needs**.

Outcome:

- Stories of use become the **foundation for building system models**.

## Key Outcome of the Lesson

By combining stories of use with conceptual modeling:

- Designers create systems that:
  - Reflect **how users understand tasks**.
  - Provide **clear guidance for developers**.

Result:

- More **efficient collaboration** between design and development teams.
