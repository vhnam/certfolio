## Levels of Detail in the Design Process

### Importance of Choosing the Right Level of Detail

- Level of detail: The amount of **information and specificity** considered during a design stage.
- Problem in design: Designers often become **lost in minor details** and forget the core user problem.
- Object-oriented design solution: Move through **structured levels of abstraction** to focus on what matters first.

Key idea:

- Start with **abstract concepts** related to user needs.
- Gradually add **more detail** as the solution develops.
- Avoid jumping directly into implementation.

## The Three Levels of Object-Oriented Design

### Essential Level

- Essential level: The **highest and most abstract level** of design.
- Focus: **Core concepts and relationships** in the user’s world.
- Perspective: What matters **conceptually to users**, not how the system works internally.

Examples of focus:

- Main objects in the system
- Relationships between objects
- Core user goals and tasks

Purpose:

- Clarify **what the system is about** before defining how it works.

### Specification Level

- Specification level: The **intermediate level** where structure and behavior are defined.
- Focus: Describe **how the system operates conceptually**.

Elements added:

- Views: How information is presented.
- Operations: Actions users or the system can perform.
- Attributes: Properties or data associated with objects.

Purpose:

- Transform abstract concepts into a **structured system design**.

### Implementation Level

- Implementation level: The **most detailed level** where the design becomes a working solution.
- Focus: Exact details required for building the product.

Typical outputs:

- Detailed prototypes
- Implementation specifications
- Final system behavior

Purpose:

- Translate the design into **a buildable and functional product**.

## Conceptual Models and Sketching

### Conceptual Models

- Conceptual model: A representation of **key objects, relationships, and behaviors** in a system.
- Role: Helps designers understand **how users mentally structure a domain**.
- Importance: Guides the design before interface details are created.

### Sketching

- Sketching: Quick visual explorations used to **test and explore possible solutions**.
- Purpose:
  - Validate conceptual models.
  - Explore interface ideas.
  - Identify usability issues early.

Key points:

- Sketches are **not part of the conceptual model itself**.
- They help test whether the conceptual model **works in practice**.

## When to Create New Conceptual Models

### Using Existing Models

You do not need to design a new conceptual model when:

- Well-known patterns already exist.
- Users are familiar with the interaction model.

Example:

- E-commerce checkout systems already have **established conceptual structures**.

Typical checkout model includes:

- Basket
- Items inside the basket
- Payment
- Delivery options

Using known models saves time and reduces usability risks.

### Creating New Models

You should create a new conceptual model when:

- Designing **entirely new interactions**.
- Introducing **innovative workflows or systems**.

Examples:

- New ways to track health activities.
- New interaction patterns for digital services.

Process:

1. Define conceptual model.
2. Create sketches of potential solutions.
3. Test ideas with users.

## Key Design Principle

Quoted verbatim:

> Designers must work at the **appropriate level of detail** during each stage of the design process.

Implications:

- Avoid focusing on UI details too early.
- Start with **conceptual understanding**.
- Progress toward **practical solutions** gradually.

## The Takeaway

- Object-oriented UI design uses **three levels of detail**:
  - Essential
  - Specification
  - Implementation

- These levels guide the transition from **abstract user needs to concrete solutions**.
- Conceptual models define the system’s **core structure**.
- Sketching helps **validate ideas and explore solutions**.
- Using the right level of detail helps designers stay focused on **solving the right problem first**.
