## Lesson 3: Object-Oriented UI Design Foundations

### Purpose of the Lesson

- Goal: Bridge the gap between **stories of use** and the **implemented product**.
- Focus: Transform user research and usage stories into **structured conceptual models**.
- Outcome: Create models that guide both **design decisions** and **software development**.

### Key Concept: Conceptual Model

- Conceptual model: A **user-centered framework** that connects what **users need** with what **developers build**.
- Role: Helps teams understand the system through the **objects that matter most to users**.
- Benefit: Aligns designers and developers around the same mental model of the product.

### Object-Oriented UI Design

- Object-oriented UI design: A design approach that structures interfaces around **real-world objects important to users**.
- Principle: People naturally think about systems through **objects and their relationships**.
- Advantage: Matches the way **developers structure software** and the way **users understand tasks**.

### Three Key Areas in This Lesson

#### Levels of Detail

- Three levels of detail: A framework to **maintain focus and clarity** throughout the design process.
- Purpose: Prevent overwhelming complexity while progressively refining ideas.

#### Building Conceptual Models

- Based on **user research and insights**.
- Translate user needs into **structured objects and relationships**.
- Helps visualize how the system should function.

#### Writing Effective Stories of Use

- Stories of use: Descriptions of **how users interact with a system to achieve goals**.
- Purpose: Capture real user behavior and needs.
- Rule: Clear stories make modeling and implementation easier.

### Supporting Tools: Unified Modeling Language (UML)

- Unified Modeling Language (UML): A standardized visual language used to **model systems and software structures**.
- Purpose: Create diagrams that describe **objects, relationships, and behaviors**.
- Tools:
  - Basic graphics tools can work for simple models.
  - Specialized UML tools improve efficiency for complex models.

- Note: Many development teams already use UML-style diagrams.

### Design Insight

Quoted verbatim:

> "The details are not the details. They make the design.”
> — Charles Eames

### Key Outcome of the Lesson

- Designers learn to **translate research and stories of use into conceptual models**.
- These models help teams:
  - Communicate clearly.
  - Build scalable systems.
  - Create **user-centered solutions that are easier to implement**.
