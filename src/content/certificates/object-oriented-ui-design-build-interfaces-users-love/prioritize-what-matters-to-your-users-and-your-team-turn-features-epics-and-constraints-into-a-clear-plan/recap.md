## Core Concepts for Managing Design Complexity

### Features

- Feature: A **characteristic of a technology solution** that delivers value to users.
- Composition: Features may include both **functional requirements** and **non-functional requirements**.
- Scope: Some features are large enough to be considered **epics**.

### Epics

- Epic: A **large functional requirement** that requires **multiple development cycles** to complete.
- Structure: Epics are typically broken down into **smaller stories of use**.

Purpose of epics:

- Organize complex functionality
- Divide large tasks into manageable implementation units

### Constraints

- Constraint: A **non-functional requirement** that determines **how a system must operate**.
- Examples include:
  - Compliance with standards
  - Performance requirements
  - Quality targets

Constraints can significantly influence design decisions and may even lead to **new functional requirements**.

## Feature Prioritization Tools

### Persona-Weighted Feature Matrix

- Persona-Weighted Feature Matrix: A tool used to evaluate **which features matter most to different personas**.
- Method: Assign **weighted scores** to personas and evaluate how valuable each feature is to them.

Benefits:

- Prioritizes features based on **user importance**
- Supports **user-centered decision-making**

### Prioritization Matrix

- Prioritization Matrix: A **cost–benefit analysis tool** used to compare **technical feasibility and user value**.

Purpose:

- Identify **high-value, feasible features**
- Balance **development effort and user benefit**

## Benefits of These Methods

### Structured Decision-Making

Using features, epics, constraints, and prioritization matrices helps teams:

- Manage design complexity
- Focus on **user value**
- Align design and development efforts
- Reduce unexpected implementation challenges

These methods provide a **systematic approach to product planning and design**.
