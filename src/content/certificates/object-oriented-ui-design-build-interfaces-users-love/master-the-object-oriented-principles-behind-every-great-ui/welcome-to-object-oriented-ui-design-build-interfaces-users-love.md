## Introduction to Object-Oriented UI Design

### What is Object-Oriented Design

- Object-oriented design (OOD): A problem-solving approach that identifies **objects from the real world** and defines how they interact to form a system.
- Objects: Real-world entities (physical, natural, or technological) that interact with each other.
- Collaboration between objects: The idea that objects “talk” to each other to accomplish tasks.

### Object-Oriented UI Design

- Object-Oriented UI Design (OOUI): Applying object-oriented thinking to **user interface design** instead of focusing on features.
- Objects in UI: Interface elements representing real-world concepts that users understand naturally.
- Object relationships: Connections between objects that define how users interact with a system.

Benefits:

- Interfaces become **more intuitive** because users think in objects.
- Designs scale better as systems grow.
- Teams gain a **shared language between designers and developers**.
- Complex requirements become easier to structure.

### Relationship with User-Centered Design

Object-oriented UI design works together with **user-centered design** by focusing on users and their goals.

Key elements involved:

- Personas
- Stories of use
- Use cases
- Persona stories

These elements help bridge the gap between **user needs and technical implementation**.

## Course Structure

### Course Focus

The course teaches how to design **user-centered solutions using object-oriented thinking**, beginning with an understanding of the users.

Starting point:

- Personas: Representations of target users and their goals.

- _Personas and User Research: Design Products and Services People Need and Want_

## Lesson Overview

### Lesson 1: Introduction to Object-Oriented UI Design

Focus areas:

- Understanding OO thinking through real-world examples (e.g., Instagram)
- Origins of object-oriented design
- Benefits of OO thinking for UX

Key concepts:

- Stories of use
- Use cases
- Persona stories

These help connect **design thinking with system implementation**.

### Lesson 2: Managing Complexity

Focus:

- Handling large and complex product requirements.

Key concepts:

- Features
- Epics
- Constraints

Tools introduced:

- Persona-weighted prioritization matrices
- Feature prioritization frameworks

Purpose:

- Help teams focus on **what matters most to users and the business**.

### Lesson 3: Conceptual Models

Conceptual models translate requirements into structured interface ideas.

Process:

1. Identify key objects in the system.
2. Define relationships between objects.
3. Organize objects into a conceptual structure.
4. Refine the model based on user goals.

Outcome:

- Clear mental models for both **users and developers**.

### Lesson 4: Visualizing and Communicating Designs

Design visualization techniques:

- Sketching
- Wireframing
- Prototyping
- Design maps

Design maps:

- Visual tools that illustrate **how users move through the interface to achieve goals**.

Benefits:

- Improve collaboration.
- Help teams communicate interface structure.
- Support developer implementation.

## UI Design Project

### Project Goal

Create a portfolio project demonstrating **object-oriented UI design skills**.

The project applies OOUI principles to either:

- A real product idea, or
- A provided sample project.

Purpose:

- Show ability to design **intuitive and scalable interfaces**.

### Project Activities

#### Lesson 1 Activity

- Identify objects and relationships within an existing product.

Goal:

- Understand how object-oriented thinking simplifies complex systems.

#### Lesson 2 Activity

- Analyze features, epics, and constraints.

Goal:

- Learn to manage complexity in real design scenarios.

#### Lesson 3 Activity

- Build a conceptual model from research insights.

Goal:

- Structure how users understand and interact with the system.

#### Lesson 4 Activity

- Visualize the solution using:
  - Sketches
  - Wireframes
  - Design maps

Goal:

- Communicate the user journey clearly.

### Learning Approach

- Activities deepen understanding through practice.
- Sample solutions are provided for reference.
- Creative solutions are encouraged.

> There are no wrong answers — experimentation and exploration are part of the design process.
