## Objects in the Real World and Design

### Definition of Objects

- Object: A “thing presented to the mind” that people can easily understand and reason about.
- Objects exist everywhere in the real world and digital environments.

Examples:

- Phone
- Chair
- Words on a screen
- Files or folders on a computer

### Object-Oriented Thinking

- Object-Oriented (OO) design: A design approach based on **objects and the actions performed on them**.
- Operations: Actions that can be performed on objects.
- Object relationships: How objects interact and connect with each other.

People naturally think about:

- Things (objects)
- What they can do with them (operations)

Because of this, object-oriented design works well for **interactive systems and user interfaces**.

### Early Object-Oriented Interfaces

Early object-based systems included:

- Xerox Star
- Apple Lisa

Characteristics:

- Users interacted with objects in the system.
- Universal operations could be applied to many objects.

Example operations:

- Open
- Move
- Delete
- Copy

These systems helped establish the **modern model of computing** used today.

## The Origins of Object Orientation

### Ivan Sutherland and Sketchpad

- Sketchpad: The first computer-aided design (CAD) system.
- Creator: Ivan Sutherland (MIT PhD student).
- Developed in the early **1960s**.

Key characteristics:

- One of the first graphical user interfaces.
- Allowed users to **create and manipulate graphical objects**.
- Objects could be moved, edited, and modified on screen.

Important insight:

- Sketchpad used **object concepts** even before object-oriented programming existed.

### Early Object-Oriented Programming Languages

Object-oriented programming languages emerged later.

Key milestones:

- Simula: First object-oriented programming language.
- Smalltalk: Popularized object-oriented design in the 1970s.

Core principle:

- Systems are built from **objects sending messages to each other**.

This model remains the foundation of modern object-oriented systems.

### Object Collaboration

Object-oriented systems rely on collaboration.

- Objects communicate with each other.
- Each object performs specific responsibilities.
- Together they form a complete system.

This mirrors **real-world systems**, where objects interact to achieve goals.

## Objects vs Pixel-Based Graphics

### Object-Based Graphics

Object-based systems treat graphics as **manipulable entities**.

Example:

- Sketchpad

Capabilities:

- Move shapes
- Resize shapes
- Edit objects independently

Objects retain their identity and properties.

### Pixel-Based Graphics

Pixel-based graphics editors manipulate **individual pixels**.

Examples:

- Microsoft Paint
- Apple Paintbrush

Characteristics:

- Images are collections of pixels.
- Shapes are not stored as objects.

Consequences:

- Editing requires modifying pixels.
- Objects cannot be manipulated independently.

## Xerox PARC and Modern Computing

### Xerox Alto and Xerox Star

The Xerox Alto and Star were early graphical systems developed at **Xerox PARC (Palo Alto Research Center)**.

Contributions:

- Graphical user interfaces
- Desktop metaphor
- Mouse interaction
- Object-based interface concepts

These systems heavily influenced:

- Apple
- Microsoft
- Adobe

### Innovations from Xerox PARC

Technologies developed or improved at Xerox PARC include:

- Computer mouse
- Local area networks (LAN)
- Laser printers
- Graphical interfaces

These innovations shaped **modern personal computing**.

## The Rise of Object-Oriented Programming

![The history of OO programming languages.](../../../../assets/images/object-oriented-ui-design-build-interfaces-users-love/master-the-object-oriented-principles-behind-every-great-ui/object-oriented-ui-design-illustrations-history-of-oo.avif)

### Motivation for Object Orientation

Object orientation became popular because it helps organize software effectively.

Advantages:

- Combines data and behavior
- Groups related functionality
- Creates clear conceptual structures

Objects allow developers to organize systems into **coherent chunks**.

### Programming Language Evolution

Important object-oriented languages include:

- Simula
- Smalltalk
- C++
- Java

Modern languages supporting object orientation include:

- Python
- Ruby
- Rust
- PHP
- TypeScript
- Kotlin
- Swift
- C++20

### C++ and the 1990s Object Boom

- Bjarne Stroustrup introduced objects into the C language.
- Result: C++.

Impact:

- Object-oriented programming became mainstream.
- The **1990s saw a massive increase in OO development**.

Object orientation remains the **dominant programming model today**.

## Object-Oriented UI Design

### Early Attempts to Apply OO to UI

In the 1990s, researchers and designers explored applying object-oriented principles to UI design.

Examples:

- Concepts published in _Object Modeling and User Interface Design_.
- Attempts to unify **user-centered design and object-oriented modeling**.

### OOUX and ORCA

Recent approaches include:

- OOUX (Object-Oriented UX)

Process model:

- ORCA

ORCA stands for:

- Objects
- Relationships
- Calls-to-Action
- Attributes

This framework helps designers structure systems around **core objects and their interactions**.

## The Desktop Metaphor

### Conceptual Model of the Desktop

![The desktop metaphor models computer systems after a physical office environment.](../../../../assets/images/object-oriented-ui-design-build-interfaces-users-love/master-the-object-oriented-principles-behind-every-great-ui/object-oriented-ui-design-illustrations-conceptual-model.avif)

The desktop metaphor models computer systems after a **physical office environment**.

Objects in the metaphor:

- Documents
- Folders
- In-tray
- Out-tray
- Wastebasket

Users understand these objects because they exist in the real world.

### Example Interactions

Typical interactions include:

- Email arrives in the in-tray.
- Documents are placed in folders.
- Files can be dragged to the wastebasket.
- Documents can be sent via the out-tray.

This metaphor made computers **easier to learn and use**.

### Limitations of the Desktop Metaphor

While successful, the desktop metaphor has limitations:

- Based on office workflows from the 1970s
- Not ideal for modern computing environments
- Some modern interfaces no longer rely on it

However, it played a crucial role in the **development of graphical interfaces and OO design**.

## Key Takeaways

- Object-oriented design organizes systems around **objects and their interactions**.
- The concept originated with **Sketchpad in the 1960s**.
- Object-oriented programming languages like **Simula, Smalltalk, and C++** expanded these ideas.
- Systems like **Xerox Star and Apple Lisa** used object-oriented principles to build graphical interfaces.
- The desktop metaphor helped users understand computers through **familiar real-world objects**.
- Today, object-oriented thinking continues to shape **software development, UX design, and modern interfaces**.
