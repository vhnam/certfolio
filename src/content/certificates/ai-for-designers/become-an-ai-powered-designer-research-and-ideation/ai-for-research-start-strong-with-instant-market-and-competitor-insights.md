## AI as a Tool, Not Intelligence

- AI amplifies input quality → clear thinking = useful output, vague thinking = noise.
- It does not understand real-world context or user reality.
- Treat all outputs as **generated possibilities**, not truth.

### Key Risks

- Hallucination: Confident but incorrect information.
- Bias: Inherited from data (pre-existing, technical, emergent).
- Overconfidence: Sounds right even when wrong.

## The Designer’s Role: From Maker to Decision-Maker

- **Before**: Create UI, flows, ideas.
- **Now**: Direct AI, shape outputs, make decisions.

### Responsibilities

- Frame the problem (what actually matters).
- Provide context (constraints, research, data).
- Define outputs (clear, usable deliverables).
- Evaluate and refine (based on real needs).

→ AI = fast executor  
→ Designer = owner of judgment

## The AI Workflow Framework

Use AI effectively by always structuring:

1. Goal — what needs to be solved
2. Context — constraints and reality
3. Inputs — research, data, assumptions
4. Output — decision-ready deliverable
5. Quality checks — validation criteria
6. Refinement — iteration process

→ If these are unclear, AI will amplify confusion.

## Prompting = Structured Thinking

- Not about clever wording.
- It’s about translating the framework into instructions.

### A Strong Prompt Includes

- Context (situation + constraints)
- Inputs (evidence, data)
- Output (clear format + purpose)
- Refinement (how to evaluate/improve)

→ Think of prompts as **design briefs for AI**.

## Four Quality Checks (Stay in Control)

### 1. Traceability

- Where did this come from?
- Unverified = hypothesis, not fact.

### 2. Context Completeness

- What’s missing?
- AI only knows what you provide.

### 3. Data vs Interpretation

- Separate facts from assumptions.
- Avoid decisions based on speculation.

### 4. Bias & Blind Spots

- Who is excluded?
- Actively check edge cases and perspectives.

## Using AI Across the Design Process

### Empathize

- Summarize, cluster research.
- Validate with real evidence.

### Define

- Generate multiple problem framings.
- Don’t lock too early.

### Ideate

- Explore many options with trade-offs.
- Don’t choose based on polish.

### Prototype

- Generate copy, states, variations.
- Ensure clarity, tone, accessibility.

### Test

- Identify usability risks.
- AI supports, not replaces users.

### Deliver

- Document decisions, changes, risks.
- Protect sensitive data.

## Key Takeaways

- Always structure: **Goal → Context → Inputs → Output → Checks → Refinement**
- Prompting = **decision-making, not phrasing**
- Validation is mandatory:
  - Source
  - Context
  - Bias
  - Fact vs interpretation

### Final Insight

- AI accelerates execution.
- Designers own **context, judgment, and outcomes**.
