## Identify Risks in Your Next Project Plan

### Lab Overview

- Purpose: Learn how to use **Gemini** to analyze project plans and identify hidden risks.
- Instructor: Owen from Google Marketing.
- Approach: Provide Gemini with project context, background information, and instructions so it can act as a strategic thinking partner.

The lab focuses on reviewing a project plan, identifying weaknesses, and improving the timeline for better outcomes.

- Risk detection: Identify potential issues before they affect project success.
- Preventive planning: Address problems early to avoid delays or failures.

#### Deeper Analysis

#### Clearer Communication

- Structured timelines: Create plans that show dependencies and connections between tasks.

## Build with AI: Risk Assessment

### Lab Overview

- Purpose: Use **Gemini** to analyze project plans, identify missing dependencies, and evaluate risks.
- Requirement: Sign into Gemini with a Google Account.
- Approach: Follow the video instructions and customize prompts to match your own project context.

This lab demonstrates how AI can help **review plans, detect gaps, and improve project timelines**.

### Step 1: Identify Missing Project Dependencies

Use Gemini to analyze planning documents and detect gaps.

Process:

1. Start a new chat with Gemini.
2. Attach project planning documents.

Example documents:

```
You are a [persona, e.g. project manager]. My company is launching a [product, service or initiative]. I've attached [document 1, e.g. a launch memo] and [document 2, e.g. a launch work-back schedule]. Are there any dependencies missing from the plan?
```

Prompt structure:

- Persona: Role Gemini should simulate (e.g., project manager).
- Project context: Product, service, or initiative being launched.
- Documents: Project planning materials provided for analysis.
- Objective: Identify missing dependencies.

Purpose:

- Detect tasks or requirements not included in the original plan.

### Step 2: Assess the Risk of Missing Dependencies

Focus on one dependency identified in the previous step.

Prompt structure:

- Missing dependency: Item highlighted by Gemini.
- Project objective: Overall goal of the project.

Example missing dependency:

```
Let's focus on [a missing dependency from Gemini’s previous response]. How do they affect the [overall project goal, e.g. the launch]?
```

Purpose:

- Evaluate how the missing dependency could impact the project.
- Understand potential risks to timelines, resources, or outcomes.

### Step 3: Generate an Updated Project Timeline

Use Gemini to build a revised timeline that accounts for the missing elements.

Prompt structure:

- Dependency: The identified missing task or requirement.
- Output: A timeline including key milestones.

Example output:

```
Create a timeline listing key milestones for [the missing dependency from the previous step]. Flag what might be missing from my current plan.
```

Expected results:

- Milestone-based project timeline
- Identification of additional missing elements
- Improved project structure

### Applying the Workflow

This AI-assisted workflow helps strengthen project planning by:

This approach can be used for many types of planning:
